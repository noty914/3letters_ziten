## 初級

定型的な画面操作をソフトが代わりに実行する仕組み。

マウスやキーボードの手順を自動化し、人的ミスを減らす目的で使われます。

## 中級

導入検討で急増しましたが、例外処理や運用保守が後追いになりやすいです。壊れたときのフォールバックもセットで決めます。

UI 操作の自動化と API 連携型の使い分け、ログ・監査、失敗時の人手フォールバックが設計の中心です。

## 上級

スケーラビリティ、クレデンシャル管理、変更耐性（UI 改修で壊れる）、CoE（卓越センター）体制が運用成熟度の論点になります。
