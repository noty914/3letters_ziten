# 用語執筆ガイド（初級/中級/上級）

このガイドは、`src/content/terms/*.md` の本文品質をそろえるための執筆基準です。

## 1. 読者レベル定義

- 初級: 略語を初めて見た人
- 中級: 定義はわかるが、実務での使い方が曖昧な人
- 上級: 設計・導入・運用の判断が必要な人

## 2. レベルごとの書き方

### 初級（短く、結論先出し）

書く順番:

1. 一言（20字前後）
2. 軽い説明（1〜2段落）
3. 任意で比喩（1段落）

ルール:

- 最初の1文で意味がわかるようにする
- つまずき論点や例外運用は書かない
- 専門語で専門語を説明しない
- 「結論が後ろ」にならないようにする

### 中級（使いどころを押さえる）

書く順番:

1. どこで使うか（典型シーン）
2. 何とセットで語られるか（隣接概念・対比）
3. ざっくり仕組み（必要に応じてつまずき）

ルール:

- 2〜5段落で収める
- 実務の会話で出る単語を優先する
- 仕様の細部は上級へ回す

### 上級（判断材料を明示）

書く順番:

1. 設計の選択肢
2. トレードオフ（性能・安全性・運用負荷）
3. 運用の詰まりどころ（障害・監視・更新）
4. 関連仕様名や標準名（必要に応じて）

ルール:

- 長くてもよいが、判断軸を先に示す
- 理想論だけで終わらせず、失敗しやすい点を入れる

## 3. コピペ用ひな形

```md
## 初級

（一言：20字前後）

（軽い説明：1〜2段落）

（任意）補足（比喩）：

## 中級

（どこで使うか：典型シーン）

（何とセットか：隣接概念・対比）

（ざっくり仕組み or よくあるつまずき）

## 上級

（設計の選択肢）

（トレードオフ：性能・安全性・運用負荷）

（運用の詰まりどころ＋関連仕様名）
```

## 4. 公開前チェック

初級:

- 1文目だけ読んで意味がわかるか
- 難語の連鎖が起きていないか

中級:

- 「使う場面」が具体的か
- 隣接概念との違いが1つ以上あるか

上級:

- トレードオフが2軸以上あるか
- 運用時の詰まりどころが書かれているか

## 5. 運用ルール（推奨）

- 執筆順は必ず「初級 -> 中級 -> 上級」
- 1回の作業単位は5〜8語
- バッチ単位でレビューし、都度全件見直しはしない
